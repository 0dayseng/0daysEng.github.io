12 /*
13  * We do NOT shadow fields that are modified when L0
14  * traps and emulates any vmx instruction (e.g. VMPTRLD,
15  * VMXON...) executed by L1.
16  * For example, VM_INSTRUCTION_ERROR is read
17  * by L1 if a vmx instruction fails (part of the error path).
18  * Note the code assumes this logic. If for some reason
19  * we start shadowing these fields then we need to
20  * force a shadow sync when L0 emulates vmx instructions
21  * (e.g. force a sync if VM_INSTRUCTION_ERROR is modified
22  * by nested_vmx_failValid)
23  *
24  * When adding or removing fields here, note that shadowed
25  * fields must always be synced by prepare_vmcs02, not just
26  * prepare_vmcs02_rare.
27  */
28 
29 /*
30  * Keeping the fields ordered by size is an attempt at improving
31  * branch prediction in vmcs12_read_any and vmcs12_write_any.
32  */
